# Rename OLLAMA_* → LLAMA_* and Make Compose Files Host-Agnostic

## Why

The project runs **llama.cpp's standalone server** (not the Ollama CLI). All references use `OLLAMA_*` env var names and mention "Ollama" in comments/docs, which is misleading. Additionally, the compose files hardcode `http://host.lima.internal:11434` (macOS/colima-only). The system needs to work on Linux (Docker/nerdctl where `127.0.0.1` IS the host) and macOS (Docker Desktop/colima) without platform-specific hardcoded hostnames.

## What Changes

- **Rename all `OLLAMA_*` env vars to `LLAMA_*`** across the entire codebase
- **Rename "Ollama" → "llama"** in comments, docs, error messages
- **Make compose files host-agnostic** — use the Makefile-provided `LLAMA_HOST` variable (defaults to `http://localhost:11434`) instead of hardcoded hostnames
- **Fix `check-ollama` target** to use llama.cpp's `/health` endpoint instead of `/api/tags`

## Capabilities

### New Capabilities
- `llama-host-agnostic-rename`: Rename `OLLAMA_*` → `LLAMA_*` across all source files, docs, and config. Make compose files use the Makefile-provided `LLAMA_HOST` variable (defaults to `http://localhost:11434`) instead of hardcoding platform-specific hostnames. Fix `check-ollama` target to use llama.cpp's `/health` endpoint.

## Impact

- **Makefile**: `OLLAMA_*` → `LLAMA_*` (5 variables), `check-ollama` → `check-llama`, endpoint fix
- **docker-compose-files/agents.yaml**: `OLLAMA_*` → `LLAMA_*` env vars, remove hardcoded hostnames
- **agents/agentics/**: all Python source and test files
- **docs/**, **README.md**, **AGENTS.md**, **hermes/skills/**, **openspec/specs/**, **agent-wiki/**
- **Not touched**: test fixtures (these are test data for the doc-sync check; they'll be regenerated by `make check-docs-sync`)
- **Archived changes**: OLLAMA references will be updated for historical accuracy
