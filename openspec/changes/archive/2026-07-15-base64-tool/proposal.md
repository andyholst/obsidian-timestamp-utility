## Why

The plugin is an Obsidian timestamp utility, but it has no text-transformation command. A small,
self-contained, non-timestamp feature — **base64 encode / decode of a message** — exercises the
same agentic pipeline (fetch/seed → generate → integrate → build → test self-correct) as the
uuid-v7 and greetings features, and gives users a genuinely useful utility: obfuscate a snippet
in a note, or decode a base64 blob pasted from elsewhere, directly from the command palette.

Unlike uuid-v7 (algorithmic, timestamp-based) and greetings (no algorithm), base64 is an
**algorithmic, non-timestamp** feature: it has real `CONTRACT_GENERATOR` bodies (`encodeBase64` /
`decodeBase64`) and a modal that wires user input to those generators. It is the third,
distinct-capability e2e proof (B3) and proves the deterministic floor handles an algorithmic
feature whose `generator_kind` is neither `uuid` nor `timestamp`.

## What Changes

- Add an OpenSpec change that drives the agentic pipeline to generate a `Base64Modal`
  (`obsidian.Modal` subclass) with two commands wired into `src/main.ts`:
  - `encode-base64-message` — opens the modal; user types a message, clicks Encode, sees the
    base64 of that message.
  - `decode-base64-message` — opens the modal; user pastes base64, clicks Decode, sees the decoded
    text (or an error Notice on invalid input).
- The `CodeIntegratorAgent` deterministic floor injects the exact `## Contract` / `## Test Contract`
  TS from THIS change's `tasks.md` (B10/B11) — no TS bodies live in Python.
- Add a new e2e integration test `test_base64_e2e_integration.py` (marker `e2e`, mirror of the
  greetings e2e) that runs the pipeline against this LOCAL change in an isolated temp dir and
  asserts the contract (command ids, modal class, encode/decode behaviour).

## Capabilities

### New Capabilities
- `base64-tool`: A base64 encode/decode command + modal generated by the agentic pipeline and
  wired into the plugin entry point. The modal accepts a message and encodes or decodes it with
  base64, displaying the result.

## Impact

- Affected code: generated `src/main.ts` + `src/__tests__/main.test.ts` (via the pipeline; never hand-authored).
- Affected tests: this change's own contract tests + the new persistent `test_base64_e2e_integration.py`.
- Affected systems: `make run-agentics CHANGE=base64-tool` + `make build-app` + `make test-app`
  (the loop-engineering gate).
