## Why

We need a SIMPLE end-to-end exercise of the whole agentic pipeline (fetch/seed → generate →
integrate → build → test self-correct loop) that is smaller and faster than the uuid-v7 feature.
A "Greetings" command + modal is the minimal Obsidian-plugin feature: one `addCommand` that opens
an `obsidian.Modal` displaying a greeting. It has no algorithmic generator (unlike uuid-v7), so it
isolates the pipeline's wiring/integration behaviour and serves as a fast smoke test of the loop.

## What Changes

- Add an OpenSpec change that drives the agentic pipeline to generate a `GreetingsModal`
  (an `obsidian.Modal` subclass) wired via `this.addCommand({ id: 'insert-greetings', ... })` in
  `src/main.ts`, plus regression tests in `src/__tests__/main.test.ts`.
- The generated command opens the modal, which displays the text "Greetings command obsidian plugin".
- The `CodeIntegratorAgent` deterministic floor injects the exact `## Contract` / `## Test Contract`
  TS from this change's `tasks.md` (B10/B11) — no TS bodies live in Python.

## Capabilities

### New Capabilities
- `greetings-modal`: A Greetings command that opens a modal displaying a greeting, generated by the
  agentic pipeline and wired into the plugin entry point.

## Impact

- Affected code: generated `src/main.ts` + `src/__tests__/main.test.ts` (via the pipeline; never hand-authored).
- Affected tests: the change's own contract tests; the persistent e2e harness may target it as an
  additional smoke feature.
- Affected systems: `make run-agentics CHANGE=greetings-modal-agentic-generation` + `make build-app`
  + `make test-app` (the loop-engineering gate).
